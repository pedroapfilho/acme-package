acme-package is a fork-ready monorepo for published TypeScript libraries. It
ships two example packages that exercise everything a real library needs:
ESM-only tsdown builds, type declarations, Vitest test suites, peer
dependencies, and Changesets releases. It also includes this docs site and a
Vite demo app that consume them the way your users would.

## What's inside

| Package                                       | Role                                                                          |
| --------------------------------------------- | ----------------------------------------------------------------------------- |
| `@acme/core`                                  | Framework-agnostic library. Zero dependencies, platform-neutral build, tests run in node. |
| `@acme/react`                                 | React adapter. React as a peer dependency, platform-browser build, tests run in jsdom. |
| `@repo/typescript-config` + `@repo/config-vitest` | Internal presets shared across the workspace. Private, never published, never renamed. |

## Architecture

The example domain is intentionally tiny: `createStore` in `@acme/core` and
`useStore` in `@acme/react`. The point is not the store — it's the
core→adapter layering most real libraries grow into: a framework-agnostic
engine with zero dependencies, and a thin framework binding that depends on it
and declares the framework as a peer. Keep that split and the rest of the
template (builds, tests, releases) keeps working as you swap in your real
code.
